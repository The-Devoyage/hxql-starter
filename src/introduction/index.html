Build dynamic apps powered by htmx, subraph, and graphql.

Near zero boilerplate. Focus on your app, not the setup.

Features

  • Dynamic Content Loading
  • Recursive Directory Routing
    • Automatically serve any file from the SSR or Assets Directory, defined in `api.toml`.
      • Ex: File located at `/src/blog/index.html` can be accessed at `https://your-site.com/blog`
    • Access a folder that does not exist will trigger a recursive search for the closest match.
      • Recursive routing allows you to use parent directories as templates, rather than having to define repetitive code.
      • Ex: Accessing Non Existent Directory `/src/blog/my-first-post` will return the closed match, `/src/blog/index.html`
      • In this example, the `index.html` file in the `blog` directory will be used as a template for the `my-first-post`.
  • Public GraphQL API
    • Define your API in `api.toml` and access it at `https://your-site.com/playground` or `https://your-site.com/graphql`
    • This configuration is used with Subgraph, a powerful tool to create dynamic APIs that connect to various data sources.
  • Handlebars Templating
    • In combination with subgraph and htmx, handlebars allows you to create dynamic content with ease.
    • Simply pass a query to the `hx-vals` property and data from your API is automatically available in your template.
